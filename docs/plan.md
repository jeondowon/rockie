# Rockie — 기획 & 작업 현황 문서

deskPet 캐릭터에 성향 발견형 성장 요소를 추가한다. 사용자는 온보딩과 일일 질문에 답하고, 누적 답변에 따라 조약돌이 4종의 원석 중 하나로 확정된다. 이후 E/I 질문과 호감도 조건을 거쳐 변성체와 보석으로 진화한다.

> 이 문서는 2026-07-11 기준 구현 상태를 코드 기준으로 재정리한 문서다.
> 실제 구현 확인 기준: `src/main/questions.js`, `src/main/evolution.js`, `src/main/store.js`, `src/main/main.js`, `src/renderer/pet/pet.js`, `src/renderer/tray/tray.js`, `test/evolution.test.js`.

## 1. 핵심 컨셉

- 캐릭터는 설치 직후 조약돌 `Rockie`로 시작한다.
- 첫 실행 시 트레이 팝업에서 운석 프롤로그 온보딩을 진행한다.
- 온보딩 질문 4개도 0→1 원석 판정 점수에 반영된다.
- 온보딩 완료 후 일일 질문이 열리고, 답변 누적에 따라 돌 종류가 확정된다.
- 돌 종류와 E/I 판정은 사용자가 직접 고르는 값이 아니라 답변 결과로 발견된다.
- 진화 단계는 역행하지 않는다. 단, 사용자는 이미 도달한 단계의 모습을 스킨으로 다시 표시할 수 있다.

## 2. 진화 단계

| --- | --- | --- | --- |

| 2단계 | 변성체 8종 | E/I 질문 6개 완료 + E/I 판정 | 완료 |

### 2.1 원석 매핑

| 돌 종류 | MBTI 기질군 | 성향 특성 | 구현 key |
| --- | --- | --- | --- |
| 화강암 | SJ | 원칙, 안정, 책임 | `granite` |
| 현무암 | SP | 즉흥, 행동, 현재 중심 | `basalt` |
| 대리석 | NF | 공감, 감성, 의미 | `marble` |
| 편마암 | NT | 논리, 전략, 구조 | `gneiss` |

### 2.2 돌별 성향 정리

| 돌 | 핵심 성향 | 자주 드러나는 판단 기준 | 강점으로 보이는 모습 | 과해질 때의 인상 |
| --- | --- | --- | --- | --- |
| 화강암 | 안정적이고 책임감 있는 계획형 | 기준, 순서, 약속, 안전성, 검증된 방법 | 맡은 일을 끝까지 해내고, 상황을 차분하게 정리한다 | 너무 조심스럽거나 변화에 늦게 반응하는 것처럼 보일 수 있다 |
| 현무암 | 즉흥적이고 에너지 있는 실행형 | 지금의 감각, 현장 반응, 기회, 빠른 시도 | 망설임 없이 시작하고, 분위기를 움직이며, 변화에 빠르게 적응한다 | 충분히 살피기 전에 움직이거나 가볍게 결정하는 것처럼 보일 수 있다 |
| 대리석 | 감정과 의미를 중시하는 공감형 | 사람의 마음, 관계의 편안함, 의미, 정서적 균형 | 주변 사람을 세심하게 살피고, 말과 분위기의 온도를 잘 읽는다 | 결정을 미루거나 감정의 영향을 크게 받는 것처럼 보일 수 있다 |
| 편마암 | 구조를 파악하는 분석형 | 원인, 논리, 효율, 전체 구조, 장단점 비교 | 복잡한 문제를 쪼개 보고, 핵심 흐름과 근거를 잘 정리한다 | 차갑거나 지나치게 따지는 것처럼 보일 수 있다 |

질문 선택지는 이 성향을 직접 노출하지 않고, 사용자의 자연스러운 선택 기준을 통해 점수로 누적한다. 예를 들어 화강암 선택지는 계획·기준·책임을, 현무암 선택지는 즉시 시도·현장감·활력을, 대리석 선택지는 마음·관계·의미를, 편마암 선택지는 구조·원리·분석을 주로 반영한다.

### 2.4 진화 연출

구현 완료:

- 진화 조건 충족 시 `pet.pendingEvolution` 저장

- 진화 완료 시 착용 중이던 이전 스킨 해제

## 3. 질문 구성

현재 질문 개수는 코드 기준으로 다음과 같다.

| 구분 | 개수 | 위치 | 판정 반영 |
| --- | --- | --- | --- |
| 온보딩 질문 | 4개 | `ONBOARDING_QUESTIONS` | 돌 점수에 반영 |
| 본 질문 | 12개 | `MAIN_QUESTIONS` | 돌 점수에 반영 |
| 본 질문 타이브레이커 | 6개 | `TIEBREAKERS` | 동점 시 돌 점수에 반영 |
| E/I 질문 | 6개 | `EI_QUESTIONS` | 외향/내향 점수에 반영 |
| E/I 타이브레이커 | 1개 | `EI_TIEBREAKER` | E/I 3:3 동점 시 반영 |

표시용 전체 진행도는 `온보딩 4 + 본 질문 12 + E/I 질문 6 = 22`로 계산한다. 타이브레이커는 표시용 전체 개수에 포함하지 않는다.

### 3.1 온보딩 질문

구현 완료:

- 운석 낙하/착지/소리/균열/조약돌 등장 흐름
- 각 장면 사이에 4개 질문 삽입
- 답변은 `traits.traitScores`에 즉시 반영
- `mainQuestionProgress`는 증가하지 않음
- 4개 모두 답해야 온보딩 완료 가능
- 온보딩 완료 후 오늘 질문 3개를 채움

### 3.2 본 질문

구현 완료:

- 본 질문 12개는 온보딩 이후 일일 질문으로 노출
- 각 선택지는 화강암/현무암/대리석/편마암 중 하나에 +1
- `mainQuestionProgress`가 12에 도달하면 원석 판정 시도
- 본 질문 문항은 기존 추상 질문이 아니라 상황 기반 질문으로 교체됨

### 3.3 E/I 질문

구현 완료:

- 원석 1단계 확정 후 E/I 질문이 노출됨
- E/I 질문은 6개
- 외향/내향 점수 중 높은 쪽으로 `evolutionVariant` 확정
- 3:3 동점이면 E/I 타이브레이커 1개를 먼저 삽입

## 4. 판정 로직

### 4.1 0→1 원석 판정

구현 완료:

1. 온보딩 질문 4개와 본 질문 12개의 돌 점수를 누적한다.
2. 본 질문 12개가 끝나면 최고점 돌을 찾는다.
3. 최고점이 하나면 즉시 1단계로 확정한다.
4. 최고점이 2개 이상이면 필요한 타이브레이커를 `todaysQuestions` 맨 앞에 삽입한다.
5. 타이브레이커 답변 후 다시 판정한다.
6. 모든 타이브레이커 후에도 동점이면 가장 최근 답변 기준으로 확정한다.

테스트 완료:

- 본 질문 완료 후 1단계 확정
- 2개 돌 동점 타이브레이커
- 3개 돌 동점 타이브레이커 순차 삽입

### 4.2 1→2 E/I 판정

구현 완료:

1. E/I 질문 6개 답변을 누적한다.
2. 외향 점수가 높으면 `extrovert` 확정.
3. 내향 점수가 높으면 `introvert` 확정.
4. 3:3이면 `eitb_01`을 `todaysQuestions` 맨 앞에 삽입한다.
5. 타이브레이커 답변 후 2단계로 확정한다.

테스트 완료:

- E/I 질문 완료 후 2단계 확정
- E/I 동점 타이브레이커

### 4.3 2→3 호감도 판정

구현 완료:

- `evolutionStage === 2 && affinityPoints >= 90`이면 3단계 확정
- 2단계 진입 시 이미 90점 이상이면 즉시 3단계로 상승
- 2단계에서 닦기/밥 주기로 90점 이상 도달해도 3단계로 상승

테스트 완료:

- 2단계 진입 즉시 3단계
- 닦기/밥 주기 후 3단계

- 답변 히스토리를 최신순으로 표시
- 이름 수정 시 사용자 이름/애완돌 이름 최초 설정 보상 반영

### 드래그 상태

기존 문서에는 직접 드래그가 미구현으로 적혀 있었으나 현재는 구현되어 있다.

- 조건: `follow` 모드에서만 동작
- 방식: 캐릭터를 200ms 이상 누르면 드래그 시작
- 놓은 뒤 3초 동안 해당 위치에 머문 뒤 다시 커서를 따라감
- 질문 카드, 진화 카드, 온보딩 잠금 중에는 드래그하지 않음

## 10. 스킨

### 구현 완료

- 스킨은 별도 코스메틱이 아니라 진화로 해금한 각 단계의 모습이다.
- 0~3단계 스킨 칸을 표시한다.
- 현재 도달 단계 이하만 착용 가능하다.
- 미래 단계는 잠금 처리한다.
- 착용 중인 단계에는 "착용중" 배지를 표시한다.
- 실제 현재 단계 칸을 누르면 원래 모습으로 복귀한다.
- 진화 완료 시 이전 착용 스킨은 해제된다.
- 펫 창과 트레이 히어로 모두 착용 스킨을 반영한다.

## 12. 부가 리액션

### 구현 완료

- 활성 앱별 말풍선

- 개발 문서, AI 도구, 리서치, 일정/할 일, 지도, 배달, 게임 등 앱·사이트별 자동 말풍선
- SNS 체류 시간별 단계형 말풍선
- 배터리 부족 구간별 sleepy/sad 리액션
- 로컬 시스템 시각 기준 시간대 리액션
- idle time 기반 장시간 사용 리액션

### 시간대 기반 리액션

구현 완료:

- 위치 권한, 날씨 API, 네트워크 호출 없이 로컬 시스템 시각만 사용
- 시간대 구분: 아침 `05:00~10:59`, 낮 `11:00~16:59`, 저녁 `17:00~21:59`, 밤 `22:00~04:59`
- 클릭 말풍선에 시간대별 문구를 추가로 섞음
- 밤 시간대에는 배터리 부족 상태가 아닐 때 기본 포즈를 `sleepy`로 표시
- 앱 실행 중 시간대가 바뀌어도 1분 단위로 상태 갱신
- 사용자 이름이 필요한 문구는 저장된 사용자 이름이 있으면 `이름님`, 없으면 `주인님`으로 출력

### 장시간 사용 리액션

구현 완료:

- Electron `powerMonitor.getSystemIdleTime()`으로 최근 입력이 없는 시간을 확인
- 5분 이상 idle이면 쉰 것으로 보고 누적 사용 시간을 초기화
- 앱이 켜진 뒤 idle 없이 활동한 시간이 45분, 90분, 120분을 넘으면 휴식 권유 문구를 클릭 메시지 풀에 추가
- macOS Screen Time 데이터 직접 접근은 사용하지 않음

### 미구현/보류

- 날씨 기반 리액션은 제외
- 앱별 멘트 추가 확장
