# Rockie — 기획 & 작업 현황 문서

deskPet 캐릭터에 "퐁퐁두두"에서 영감을 받은 성향 발견형 성장 요소를 추가한다.
사용자는 정해진 주기로 질문에 답하고, 쌓인 답변에 따라 캐릭터가 4종의 돌 중 하나로 결정되며,
이후 시간과 활동이 쌓이면 각 돌은 단계적으로 변성·진화한다.

> 이 문서는 기획 스펙(2장 이후)과 **현재 작업 현황**을 함께 담는다.
> 아래 "현재 진행 상황 / 남은 작업"이 최신 요약이고, 각 장의 구현 상태·미구현·보류 항목이 세부 백로그다.
> 진화 시스템의 최신 스펙은 `docs/evolution.md`·`docs/update.md`(v2)를 정본으로 본다.

## 현재 진행 상황 (2026-07-11)

핵심 슬라이스(질문 → 돌 확정 → 진화 → 트레이 UI → 호감도 → 하트 → 시스템 모니터)는 동작한다.

- **진화 v2 반영** — 0~3단계(조약돌 → 돌 확정 → 변성체 e/i → 보석) 전환, GIF 경로 레벨(level0~3) 인식, 매일 8시 질문 갱신, 답하기 버튼, 인앱 초기화. (정본 스펙: `docs/evolution.md`·`docs/update.md`)
- **판정·질문 흐름** — 본 질문 12개 집계 → 돌 확정 → GIF 전환. 질문은 트레이 "나의 애완돌"의 콜아웃으로 능동 접근하고, 실제 답변 카드는 캐릭터 옆 오버레이에서 연다. 트레이 배지와 8시 OS 배너 알림은 분리되어 동작한다.
- **트레이 팝업** — 4화면(MENU/PET/SYSTEM/SETTINGS) 이식 + 설정 전 항목 배선(자동실행·질문알림·효과음값·펫 위치 3택·크기 3택·초기화). 이름 편집, 성향 요약, 진행도 힌트, 답변 히스토리 바인딩. PET 히어로는 현재 펫 창의 스프라이트 포즈를 동기화하고, `docs/evolution.md`의 단계별 성향 설명을 수동 개행으로 표시한다.
- **호감도 트랙(일부)** — 상한 100, 사용자 이름 최초 지정 +5, 애완돌 이름 최초 지정 +5, PET 게이지·수치·하트(5칸=100) 실제값 반영. 5단계 레벨명(낯가림~각별, 균등 20점)·타이틀바 pip 배선.
- **돌보기·하트** — 트레이 닦기/밥 → 하트 오버레이 발동, 캐릭터별 위치 튜닝(HEART_OFFSET) 완료.
- **스킨 착용/해제** — 스킨은 진화로 언락한 각 단계의 모습 그 자체다(진화와 무관한 별도 코스메틱 스킨은 없음). 트레이 스킨 그리드(0~3단계)에서 도달한 단계를 골라 데스크톱 펫·트레이 히어로를 그 단계 형태로 표시하고, 잠금(도달 전)·착용중 배지·현재 단계 복귀까지 배선 완료(`setActiveSkin`·`evolution:set-skin`→`pet:skin-change`).
- **진화 전환 검증** — `node --test` 기반 테스트 추가. 0→1, 본 질문 2개·3개 동점 타이브레이커, 1→2, E/I 동점 타이브레이커, 2단계 진입 즉시 3단계, 닦기/밥 2→3, 2단계 이후 배지 제거 검증.
- **진화 애니메이션** — 단계 전환 시 pending 상태 저장, 말풍선 예고 → 애완돌 클릭 → 카드 내 fade 전환 → 카드 닫기 완료 문구 흐름 적용.
- **시스템 모니터** — `systeminformation` 기반 CPU/RAM/디스크/배터리/네트워크 조회, SYSTEM 화면 진입 중 2초 폴링, 게이지·상세 드롭다운·색상 구간·CPU 부하 기반 기분 카드 배선. 반응 카드의 작은 펫은 현재 진화/포즈 스프라이트와 연동하고, CPU 부하 구간에 따라 회전 속도가 바뀐다.

## 남은 작업 (통합 백로그)

세부·근거는 각 장의 구현 상태·미구현·보류 항목 참고.

- **진화**: 시각 자산은 level0~3 전부 완료. 추가 진화 백로그는 실제 플레이 테스트 후 보강
- **호감도**: 레벨 구간표·레벨명·pip 배선 완료. 레벨별 언락 아이템·기타 상호작용 상한·쿨다운 수치는 **구현 안 함(2026-07-11 결정)**
- **트레이 UI**: 질문 콜아웃→모달은 **트레이에서 안 열기로 결정(스킵)**, 인벤토리는 **보류**. (스킨·하트·도트 폰트 로컬 번들은 완료)
- **시스템 정보(§10)**: 기본 수치 조회·표시와 부하별 SYSTEM 반응 카드 펫 회전 속도 연동은 완료. 남은 작업은 폴링 비용 실사용 검증(메뉴 상단 요약 라인은 제거됨)
- **채팅(§11)**: **보류(2026-07-11 결정)** — 모델 선택·node-llama-cpp 연동·말투 프롬프트·요약 압축
- **부가 리액션(§12)**: 활성 앱별 말풍선·SNS 체류 단계형 멘트·배터리 부족 리액션은 구현. 시간대/날씨 멘트는 보류
- **드래그/위치(§13)**: 트레이 설정의 따라오기/좌하단/우하단 고정과 크기 3단계는 구현. 드래그로 직접 위치 이동은 미구현
- **리소스 구조(§7)**: PNG 시퀀스+Canvas 전환·팔레트 스왑·레이어 합성 — 모자 착수 시
- **BM/배포(§15)**: 후원 링크·아이템 유료화·App Store 대응 — 보류

## 1. 개요

- 캐릭터는 메뉴바 트레이 아이콘 → "나의 애완돌" 메뉴에서 확인 가능
- 캐릭터 디자인은 기존 픽셀(원 + 눈 두 개) 실루엣을 유지하고, 색상·미세한 표면 요소로만 구분
- 진화 단계는 되돌아가지 않는다 (역행 없음)

## 2. 성향 ↔ 돌 매핑

MBTI의 4대 기질(Temperament) 이론을 기반으로 4종의 돌을 배정한다.

| 돌 종류 | MBTI 기질군   | 성향 특성                    | 실제 지질학적 서사                                             |
| ------- | ------------- | ---------------------------- | -------------------------------------------------------------- |
| 화강암  | SJ (관리자형) | 원칙주의, 안정 지향, 책임감  | 마그마가 지하 깊은 곳에서 천천히 식어 형성. 단단하고 결이 고름 |
| 현무암  | SP (탐험가형) | 즉흥적, 행동 지향, 현재 중심 | 용암이 지표에서 급격히 식어 형성. 다공질, 거친 표면            |
| 대리석  | NF (외교관형) | 이상주의, 공감, 감성 중심    | 석회암이 열·압력을 받아 변성. 매끄럽고 예술적                  |

## 3. 진화 4단계 (v2)

> v2에서 3단계 → **4단계(0~3단계)**로 확장했다. E/I(외향/내향) 결정 시점이 기존 "2단계 세부 변형"에서 **1→2 전환 조건**으로 이동했고, 최종 진화체(보석)가 3단계로 분리됐다. 아래는 요약이며, 결정값·데이터 필드·의사코드 정본은 `docs/update.md`, 단계별 서사·진화 근거는 `docs/evolution.md`를 본다.

| 단계  | 명칭                 | 진입 조건                      | 시각 자산                                                             |
| ----- | -------------------- | ------------------------------ | -------------------------------------------------------------------- |
| 0단계 | Rockie (디폴트)      | 앱 설치 직후                   | 무채색 원형 캐릭터 (level0/rockie)                                   |
| 1단계 | 원석 (4종)           | 본 질문 12개 완료 + 판정 확정  | 화강암·현무암·대리석·편마암 (level1)                                 |
| 2단계 | 변성체 (4종·E/I 갈래) | E/I 질문 12개 완료 + 판정 확정 | 페그마타이트·에클로자이트·코런덤 대리석·미그마타이트, 각 e/i (level2) |
| 3단계 | 보석 (8종)           | 호감도 90 이상 도달            | 3.1 매핑 (level3)                                                    |

> 단계는 역행하지 않으며, 각 전환은 조건 충족 즉시 판정한다(지연 없음).

### 3.1 계열별 진화 매핑 (원석 → 변성체 → 보석)

E/I는 1→2 전환 시 확정되고, 3단계 진입 시 그 값으로 자동 매핑된다(별도 판정 없음).

| 원석(1단계) | 변성체(2단계) | 3단계 외향(E)            | 3단계 내향(I)     |
| ----------- | ------------- | ------------------------ | ----------------- |
| 화강암      | 페그마타이트  | 토파즈                   | 아쿠아마린        |
| 현무암      | 에클로자이트  | 다이아몬드(브릴리언트컷) | 다이아몬드(원석)  |
| 대리석      | 코런덤 대리석 | 파티사파이어             | 루비              |
| 편마암      | 미그마타이트  | 라브라도라이트           | 문스톤            |

> 변성체 지질학적 근거: 화강암→페그마타이트(거대 결정 성장), 현무암→에클로자이트(고압 결정화), 대리석→코런덤 대리석(루비·사파이어핵), 편마암→미그마타이트(부분 용융 줄무늬). 다이아몬드는 v2에서 현무암 계열 3단계 보석으로 편입됐다(기존 "숨겨진 5번째 등급" 이스터에그 설정 폐기).

### 3.2 단계 전환 조건

| 전환                  | 조건                                                                     |
| --------------------- | ------------------------------------------------------------------------ |
| 0→1 (원석 확정)       | 본 질문 12개 완료 → 4종 돌 점수 집계·확정 (동점 시 타이브레이커)          |
| 1→2 (변성체·E/I 확정) | E/I 질문 12개 완료 → 외향/내향 집계·확정 (6:6 동점 시 E/I 타이브레이커 1개) |
| 2→3 (보석)            | `evolutionStage == 2` 이고 호감도 `affinityPoints >= 90`. 호감도 증가(닦기/밥) 직후에만 체크 |

- 호감도는 단계와 무관하게 상시 누적된다. 단 2→3은 2단계 진입이 전제라, 2단계 이전에 90을 넘겨도 즉시 진화하지 않고 2단계 진입 순간 90 이상이면 그때 즉시 3단계로 판정한다.
- E/I는 1→2 전환의 조건으로 상승했다. v1의 시간·활동 게이트("확정 45일 + 누적 20답변", `canEvolveToStage2`)는 폐기됐다.
- 질문 노출은 하루 최대 2개를 사용자가 트레이 "새로운 질문에 답하기"로 능동적으로 답하는 방식이며, 매일 08:00 갱신·스킵 없음(update.md §3).

### 3.3 사용자 체감 타임라인

| 경과                                       | 상태                                  |
| ------------------------------------------ | ------------------------------------- |
| 설치 → 약 1~2주                            | 본 질문 12개 소진 → 원석 확정 (1단계) |
| 이후 약 1~2주                              | E/I 질문 12개 소진 → 변성체 (2단계)   |
| 호감도 90 도달 (닦기·밥 매일 수행 시 최소 23일) | 보석 (3단계)                     |

### 3.4 ⬜ 미구현·보류

- 시각 자산은 level0~3 전부 제작 완료(원석 4·변성체 8·보석 8). 로직 배선의 남은 부분은 상단 "남은 작업" 백로그 참고.
- 상태 전이 다이어그램(질문 큐 상태 시각화) — 제작 여부 미확정.

## 5. 판정 로직

1. 본 질문(12개) 답변 완료 시 4개 돌의 누적 점수를 집계한다.
2. 최고 점수 돌이 하나뿐이면 그대로 확정한다.
3. 2개 돌이 동점이면, 해당 쌍의 타이브레이커 질문 1개를 추가로 제시하고 결과에 따라 확정한다.
4. 3개 이상 동점이면, 동점 그룹 내 모든 쌍의 타이브레이커를 순서대로 제시해 재집계한다. 그래도 동점이면 가장 최근 답변 기준으로 확정한다.
5. 확정된 돌 종류는 이후 변경되지 않는 고정값으로 저장한다.

의사코드:

```
maxScore = max(화강암, 현무암, 대리석, 편마암)
tied = 점수가 maxScore인 돌들의 목록

if tied.length == 1:
    확정(tied[0])

else if tied.length == 2:
    질문 = 타이브레이커[tied[0], tied[1]]
    답변에 매칭된 돌 +1점
    확정(그 돌)

else if tied.length >= 3:
    for each pair in combinations(tied, 2):
        질문 = 타이브레이커[pair]
        선택된 돌 +1점
    재집계 후 최고점 확정
    (그래도 동점이면 → 마지막 본 질문 답변 기준으로 확정)
```

7. **스킵 없음**: v1의 "지금은 패스" 옵션은 제거됐다(모든 질문 응답 필수).
8. **타이브레이커**: 동점 판정 시 다음 `todaysQuestions` 맨 앞에 삽입해, 사용자가 답하는 시점에 확정한다.

## 7. 캐릭터 리소스 구조 (확장성 관련 결정 사항)

- 향후 모자 등 아이템 추가, 돌 종류·진화 단계 조합 확장을 고려하면 현재의 "상태별 개별 GIF" 방식은 조합이 기하급수적으로 늘어나 비효율적

- 리팩터링 시점은 모자 등 아이템 기능을 실제로 착수할 때가 적기

| 스킨                 | 디폴트(0단계) 스킨 + 도달한 단계(1단계 원석, 2단계 변성체, 3단계 보석)별 스킨을 썸네일 그리드로 표시. 사용자가 원하는 스킨을 자유롭게 선택해 착용 가능 (순수 표시용, 진행 상태에는 영향 없음). 아직 도달하지 못한 단계는 실루엣만 흐리게 표시된 "잠김" 상태로 노출 |
| 답변 히스토리 (선택) | 지난 질문·답변을 스크롤로 다시 볼 수 있는 섹션                                                                                                                                                                                                                 |

### 8.2 스킨 시스템 설계 원칙

- 스킨 선택은 구현됐다. 원칙은 "표시(display)" 값이며, 실제 진행 상태(단계, E/I 변형)와는 독립적으로 `pet.activeSkinStage`에 저장한다. 진화하면 착용 스킨을 벗고 새 형태를 보인다(`completePendingEvolution`).
- 스킨은 진화로 언락한 각 단계의 모습 그 자체다. 진화와 무관한 별도 코스메틱 스킨(예: 계절/테마 스킨)은 계획에 없으므로, 각 칸이 그 단계의 진화 자산을 그대로 쓰는 것이 정상이다(전용 스킨 리소스 개념 없음). 트레이 스킨 그리드는 0~3단계 기준(`data-stage` 0/1/2/3)으로 정리됐다.
- 착용 가능한 스킨은 현재까지 도달한 단계(`stage >= 칸`)까지만 해당하며, 도달하지 못한 미래 단계는 잠금 커버로 미리보기만 제공한다. 착용 중인 칸에는 "착용중" 배지를 표시한다.

### 8.3 구현 상태와 남은 작업

디자인 이식은 완료됐고(화면 전환·나의 애완돌·설정 전 항목·이름 편집·성향 요약·답변 히스토리·PET 히어로·SYSTEM 모니터 등은 "현재 진행 상황" 참고), 아래는 아직 기능 연결이 남은 항목이다.

- **한글 도트 폰트** — ✅ 완료 — **Galmuri 확정**. 2026-07-11 CDN(jsdelivr) 의존을 로컬 번들로 전환(오프라인 대응). 실제 사용하는 `Galmuri11`·`Galmuri14` Regular woff2만 `assets/fonts/`에 포함하고 `assets/fonts/galmuri.css`의 `@font-face`로 로드(Bold/Condensed·Galmuri9/7/Mono는 미사용이라 제외). 로드 실패 시 `monospace` 폴백
- **메뉴 화면**: 상단 시스템 상태 라인은 **제거됨**(2026-07-10, MENU 정적 라인 삭제 — 시스템 정보는 SYSTEM 화면으로 일원화)
- **PET · 인벤토리**: **보류(2026-07-11)** — 모자 등 아이템 자산 미착수(§7)라 장착/언락 UI도 함께 연기
- **PET · 스킨**: ✅ 완료 — 스킨 선택/착용 시 데스크톱 펫·트레이 히어로 반영, 진화 단계별 언락, 잠금/착용중 상태, 0~3단계 라벨 정리까지 구현. 스킨은 진화로 언락한 단계 형태 자체라 전용 리소스가 따로 필요 없음(남은 작업 없음)
- **PET · 히어로/기분**: 현재 스프라이트 포즈 동기화와 성향 설명 표시 완료. 히어로 애니메이션은 기존 bob 유지. SYSTEM 반응 카드의 작은 펫은 현재 스프라이트를 52px로 표시하고 CPU 부하에 따라 회전 속도가 변한다.
- **질문 콜아웃/모달**: **트레이에서 열지 않기로 결정(2026-07-11 스킵)** — 답변은 캐릭터 옆 오버레이 흐름을 정본으로 유지
- **설정 · 효과음**: ✅ 완료 — 펫 렌더러 `shared/sound.js`가 Web Audio로 클릭·돌보기(밥/닦기)·진화 시 8비트풍 효과음 합성(오디오 파일 없음). `soundEnabled` 토글로 on/off, 트레이 변경 시 `pet-settings`로 즉시 반영

## 9. 호감도(관계) 트랙 / 구현 완료

호감도는 진화(성향 판정)와 독립된 별도 관계 트랙이다. `affinity.affinityPoints`(0~100)에 적립한다.
(상한 100·사용자 이름 최초 지정 +5·애완돌 이름 최초 지정 +5·PET 게이지 배선은 완료 — "현재 진행 상황" 참고.)

**획득 방식 (v2 확정, `update.md` §6)**
- 애완돌 **닦아주기 +2 · 밥 주기 +2**, 각 하루 1회, 매일 08:00 리셋(하루 최대 +4).
- 호감도 90 도달 시 2→3 진화 트리거(§3.2). 단계와 무관하게 상시 누적된다.

**레벨 구간 (확정·배선 완료)**
- 균등 20점 5구간: **낯가림 0–19 · 서먹 20–39 · 친근 40–59 · 살가움 60–79 · 각별 80–100**. 표시 전용이며 2→3 진화는 raw 90점 사용.
- 배선: `tray.js` `affinityLevel(points)`가 레벨(1~5)·명칭을 계산 → PET 호감도 섹션에 레벨명, 타이틀바 pip 5칸을 레벨만큼 점등(`renderAffinity`). 레벨별 언락 아이템은 **구현 안 함(2026-07-11 결정)**.

## 10. 시스템 정보 표시 (RunCat 영감) — 구현 완료

RunCat처럼 시스템 정보를 캐릭터/화면으로 표시하는 기능. 트레이 SYSTEM 화면의 기본 데이터 조회·표시와 반응 카드 연동은 구현됐다.

- **구현됨** — `systeminformation`으로 CPU/RAM/디스크/배터리/네트워크 값을 조회하고, SYSTEM 화면이 열려 있는 동안만 2초마다 폴링한다. 조회 실패 시 이전 값을 유지한다.
- **표시됨** — 각 항목의 수치·게이지·상세 드롭다운·초록/노랑/빨강 구간 색상, 메모리/디스크 GB, 배터리 상태/헬스/사이클, 네트워크 IP·업/다운 속도.
- **반응 카드** — CPU 부하 기준으로 "새근새근 · 여유" / "달그락 달그락 · 활동적" / "데굴데굴 · 바쁨" 문구와 회전 속도를 전환한다. 이미지는 현재 펫 스프라이트를 52px 크기로 표시한다. 속도는 여유 12s, 활동적 3.2s, 바쁨 0.35s 구간으로 체감 차이를 크게 둔다.
