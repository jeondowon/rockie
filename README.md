# Desktop Pet (MVP)

작은 캐릭터가 화면 위에서 마우스를 따라다니며, 클릭하면 반응하고, 현재 활성화된 앱/창 제목에 따라 상황별 말풍선 메시지를 표시하는 Electron 데스크톱 앱입니다.

## 1. 설치 및 실행

```bash
npm install
npm start        # 일반 실행
npm run dev      # 파일 저장 시 렌더러 자동 새로고침 (개발용)
```

## 2. 파일 구조

- `main.js` : 전체 화면을 덮는 투명·항상 위 창 생성, 전역 커서 위치 전달(16ms 주기), 활성 창 정보 폴링(3초 주기), 개발용 자동 리로드
- `preload.js` : 렌더러 ↔ 메인 프로세스 간 안전한 IPC 브릿지(`window.petAPI`)
- `index.html` / `style.css` / `renderer.js` : 캐릭터 이동·방향 전환, 말풍선, 클릭 반응, 메시지 매핑 로직
- `left.gif` / `right.gif` : 좌/우 방향 캐릭터 스프라이트
- `package.json` : 의존성 정의 (`electron`, `active-win`)

## 3. 동작 방식

1. 화면 전체 크기의 투명 창을 하나 만들고, 기본적으로 클릭이 뒤쪽 앱으로 통과되도록 설정합니다.
2. 창이 클릭 통과 상태라 렌더러가 마우스 이벤트를 직접 받을 수 없으므로, 메인 프로세스가 16ms마다 전역 커서 위치를 렌더러로 전달합니다.
3. 캐릭터는 화면 하단의 고정 라인에서 마우스 반대편에 서서, 그림 가장자리를 커서에 부드럽게(easing) 붙이며 따라다니고, 마우스가 있는 쪽을 바라봅니다(`left.gif`/`right.gif` 전환).
4. 마우스가 캐릭터 위에 있을 때만 일시적으로 클릭을 받도록 전환합니다(그 외 영역은 클릭 통과).
5. 클릭하면 랜덤한 짧은 반응 메시지를 보여주고 잠시 멈춥니다.
6. 3초마다 현재 활성 창(앱 이름, 창 제목)을 확인해서, `renderer.js`의 `rules` 배열에 정의된 패턴과 일치하면 해당 메시지를 말풍선으로 띄웁니다.
   - 예: VSCode/IntelliJ/Cursor 등 코드 에디터 → "집중모드 ON!"
   - 예: 창 제목에 "YouTube" 포함 → "즐감하세요~"

## 4. 참고 및 주의사항

- macOS에서는 `active-win`이 활성 창 정보를 읽기 위해 **손쉬운 사용(Accessibility) 권한**을 요구합니다. 앱 최초 실행 시 시스템 설정에서 권한을 허용해야 정상 동작합니다.
- Windows에서는 별도 권한 없이 바로 동작합니다.
- `renderer.js`의 `rules` 배열에 패턴만 추가하면 원하는 앱/사이트에 대한 메시지를 자유롭게 확장할 수 있습니다.
- 캐릭터를 교체하려면 `left.gif` / `right.gif`를 바꾸고, 필요 시 `style.css`의 `.character` 크기(현재 128px)와 `renderer.js`의 `CHAR_SIZE`·`SPRITE_MARGIN` 값을 함께 맞춰주면 됩니다.

## 5. 향후 확장 아이디어

- 클릭 시 캐릭터를 드래그해서 위치 이동
- 트레이 아이콘으로 온/오프, 캐릭터 교체 메뉴 제공
- 앱별 메시지 규칙을 JSON 설정 파일로 분리해서 코드 수정 없이 편집 가능하게 만들기
- 마우스를 따라오는 대신 자유롭게 배회하는 모드 등 여러 행동 패턴 추가
